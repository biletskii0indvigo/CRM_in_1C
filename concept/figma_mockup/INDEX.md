## ⛔ Критическое замечание

**Это НЕ скриншоты 1С-форм.** Это **концептуальный Figma-макет**, описывающий желаемую UX-структуру без привязки к реальной платформе 1С:Предприятие.

### Что есть в файле

- Описание разделов Figma-проекта (00_Cover, 01_Product_Map, 02_Design_System, …)
- Карта бизнес-сущностей (абстрактная)
- User Flow для CJM
- Описание дашбордов

### Чего НЕТ

- Ни одного скриншота реальной управляемой формы 1С
- Ни одного скриншота из текущей 1С:Медицина.Поликлиника заказчика
- Ни одного скриншота прототипа 1С, обещанного к 22.05 (Meet 8)
- Нет соответствия между блоками blueprint и объектами 1С (документами, справочниками)

## Проблемы

| Проблема | Влияние |
|----------|---------|
| **Имя папки вводит в заблуждение** — «forms and screen» подразумевает реальные формы | Команда ожидает увидеть 1С-формы, а получает абстрактный UX-документ |
| Версия — `v_1` (на 21.05.2026) | Не обновлялся после Meet 5–8 (видимость тегов, личные теги, ортопедия как отд. заказ) |
| Не привязан к функциональным требованиям `docs/FR-XXX` | Невозможно понять, какой FR обслуживает каждый макет |
| Является копией `demo app/version 2/docs/figma_blueprint_crm_1_c_medicine_module_v_1.md` | Дублирование |

## Что должно быть на месте этой папки

1. **Скриншоты текущих форм 1С:Медицина.Поликлиника** заказчика (что есть сейчас, AS-IS).
2. **Эскизы новых форм 1С** (Карточка пациента CRM, Заказ CRM, Рабочее место куратора) — те самые «прототипы к 22.05», обещанные на Meet 8.
3. **Соответствие** между формой и FR-ID.
4. **Список реквизитов** на каждой форме с указанием обязательности / типа.

---

## Рекомендации

1. **Переименовать папку** в `concept/figma_mockup/` чтобы не вводить в заблуждение.
2. Создать новую папку `blueprint/1c_forms/` для реальных макетов 1С-форм.
3. После Discovery 1С (см. `PROBLEM_ANALYSIS.md` §П-1) — добавить AS-IS скриншоты.
4. После создания прототипа Meet 8 — добавить экспортированные формы (PNG из конфигуратора 1С).
5. Связать каждый макет с конкретным FR-ID и Сюжетом тест-кейса.
