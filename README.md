## О проекте

CRM-модуль автоматизирует полный жизненный цикл пациента — от первичного обращения до повторных циклов лечения в сети клиник «Индвиго». Реализуется как расширение `.cfe` к типовой конфигурации 1С:Медицина.Поликлиника без изменения типовой конфигурации.

### Ключевые функциональные блоки (MVP)

- **Карточка пациента** — единый профиль 360° с антидубликатом (FR-001)
- **Заказ CRM** — базовая единица работы; 5 типов исходов сделки (FR-002, FR-024)
- **Воронка с 9 статусами** — от «Записан» до «Завершён» / «Отказ» (FR-003)
- **Цветные теги** — иногородний, ДМС, возраст и другие (FR-004)
- **ABC-сегментация** — A/B/C + ВИП + реанимация (FR-005)
- **Система задач** — структурированные задачи вместо комментариев (FR-006)
- **Триггеры** — +3 дня после приёма, МРТ-триггер, следующий контакт (FR-007, FR-027, FR-009)
- **Роль помощника куратора** — рабочее место + ограничения доступа (FR-011, FR-012)
- **Ортопедический заказ** — отдельный сценарий (FR-025)
- **Правило непрерывности** — частичный выкуп: +3/+10 дней, тот же переговорщик (FR-026)
- **Замещение и перераспределение** при отпусках и увольнениях (FR-014, FR-015)

### Ключевые решения (Meet 9, D28–D35)

| Решение | Содержание |
|---------|-----------|
| D28 | 5 типов исходов: полный выкуп / частичный выкуп / пауза / только диагностика / отказ |
| D29 | Частичный выкуп: звонит тот же куратор, что вёл переговоры |
| D30 | Местные пациенты: дожим +3 дня; иногородние: +10 дней |
| D31 | Ортопедия: отдельный заказ, всегда полный выкуп, финальная консультация не нужна |
| D32 | Мониторинг дорогостоящих процедур (ботулинотерапия, аджови) |
| D33 | Автотриггер: звонок куратора после МРТ |
| D34 | Промежуточная консультация 20 мин — пока без записи в номенклатуру |
| D35 | Предопределённые теги — видны всем; личные теги — не передаются замещающим |

## Правила работы с репозиторием

1. **Не изменять MoM** — протоколы встреч не редактируются после публикации.
2. **Источник истины** — `MoM/Summary_Meet_0_to_9.md` имеет приоритет над `docs/`.
3. **Именование объектов 1С** — префикс `CRM_` для всех объектов расширения.
4. **OQ** — каждый нерешённый вопрос оформляется как OQ-XXX в `docs/OPEN_QUESTIONS.md`.
5. **152-ФЗ** — скриншоты из 1С:Медицина не публикуются в репозиторий без деперсонализации.
