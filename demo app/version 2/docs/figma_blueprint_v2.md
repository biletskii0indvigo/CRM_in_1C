## 1. Назначение документа

Blueprint v1 описывал высокоуровневую структуру Figma-проекта и общие UX-принципы. Blueprint v2 — это **трассировочный документ**: фиксирует, что именно реализовано в HTML-прототипе (`CRM Институт Движения.html`), как экраны соотносятся с FR-001…FR-023, какие сценарии покрыты, а какие остаются в Out-of-Scope текущей итерации.

Использовать как:
- источник истины при сверке прототипа с ТЗ;
- основу для Figma-страниц `04_Wireframes` и `05_UI_Screens`;
- чек-лист для приёмки Фазы 1 (MVP).

## 2. Карта экранов прототипа

Прототип построен как single-page application в стиле управляемого приложения 1С:Предприятие. Навигация — через левое меню (раздел «Маркетинг» в нижнем уровне общего меню МИС) и систему вкладок (TabBar).

### 2.3 Скриншоты по экранам — где смотреть

Все экраны живут в одном файле `CRM Институт Движения.html`. Открываются:
- через левое меню → подпункт «Маркетинг»
- через переключатель ролей (открывает целевой workspace)
- через двойной клик / гиперссылку в любом списке (открывает карточку документа в новой вкладке)

### 4.2 Список «Пациенты (CRM)» — основной реестр

**FR-покрытие:** FR-001, FR-004, FR-005, FR-017, FR-020.

**Колонки** (17 шт., сортировка по любой):
1. ☑ — мульти-выбор
2. № карты (link → карточка)
3. Дата входа
4. Пациент (link + значки: 📵 «Не связываться», ДМС-pill, 🎫 абонемент N/M)
5. **ABC** — pill A/B/C/ВИП/Реанимация
6. **Цикл** — №N · Тип цикла
7. Акт. заказов (счётчик)
8. Сумма по заказам
9. Оплачено
10. К оплате
11. След. контакт (красный если дожим день 7+)
12. Куратор (с цветной точкой)
13. Комментарий + [Дожим: день N]
14. Статус
15. Тип
16. Приоритет (А/В/С)
17. Тип ведения, Психотип, Гео, Источник

**Фильтр-панель тегов:** 7 групп (Тип / Ведение / Гео / Психотип / Источник / Формат / Дожим) с pills-чипами.

### 4.3 Канбан «Воронка продаж»

**FR-покрытие:** FR-003 (стадии пациента).

**6 колонок:** Лид → Записан → В лечении → Финальная → Ожидание → Реактивация.

**Карточка:**
- ФИО + крупный
- Источник · дата · сумма к оплате
- **Полоска заказов:** «N/M заказов · Σ ₽» (FR-002)
- Цветные pills: тип пациента, ведение, психотип, дожим
- Куратор внизу с цветной точкой

**Drag-and-drop:** между колонками, при drop — toast с переходом старая→новая.

### 4.4 Утреннее планирование

**FR-покрытие:** FR-013 (частично).

**Layout:** двухколоночный — слева пул нераспределённых, справа 3 дорожки кураторов.

**Действия:**
- ✦ Авто-распределить (round-robin)
- ↺ Сбросить
- 📨 Уведомить кураторов
- Drag-and-drop пациентов между пулом и дорожками

**Метрики сверху:** 4 stat-box (Активных / Без куратора / Финальных / Кураторов).

**Гэп:** счётчик нагрузки куратора в часах (30–40 мин × N пациентов) — отсутствует.

### 4.5 Задачи кураторов

### 4.7 Карточка пациента

**FR-покрытие:** FR-001, FR-005, FR-008, FR-016, FR-017.

**Структура (по разделам):**

1. **Шапка**: аватар + ФИО + № карты + стадия + статус + тип
2. **Командная панель**: Записать и закрыть (синяя, FR-008 — disabled пока не 6/6) | Записать | Оформить отказ | +Заказ | +Задача | Позвонить | Синхронизировать
3. **Показатели по заказам** (5 метрик)
4. **Заказы пациента** — таблица всех заказов с возможностью двойного клика → карточка заказа
5. **Атрибуты пациента**:
   - Ответственный куратор
   - Дата оплаты (план.)
   - **ABC-сегмент** (FR-005)
   - **Цикл** (FR-017): №N + тип (Острый/Хронический/Профилактический/Реабилитационный)
   - Приоритет А/В/С
   - Психотип
   - География, Источник
   - **Флаги** (FR-004): ДМС, Не связываться, абонемент-индикатор
6. **Блок «Ведение»** (FR-008) — выделен жёлтой рамкой:
   - Тип ведения (4 опции)
   - Подтип ведения (зависит от типа)
   - Дата следующего контакта
   - Формат контакта (визит/процедура/диагностика)
   - Комментарий куратору (до 280 символов)
   - **Ответственный куратор** ← 6-е поле, добавлено в v0.1.0
   - Счётчик «X из 6» + блокировка
   - При полном заполнении → зелёный info с авто-задачей на −7 дней (FR-009)
7. **Вкладки внизу**: Задачи пациента / Дополнительно (договор, ИНН, ОМС/ДМС) / Стратегия 0-3-7-14-30

### 4.8 Карточка заказа

**FR-покрытие:** FR-002, FR-003 (status9), FR-017 (тип приёма).

**Главные блоки:**
- Шапка: № заказа + дата + ссылка на пациента + status9 pill
- **Номенклатура** (таблица услуг с qty/price/sum + «Итого»)
- **Реквизиты**: номенклатурная группа, мед. орг., **status9** dropdown, исполнитель, источник, оплата
- **Задачи по заказу** — таблица task'ов с двойным кликом
- Сайдбар справа: Финансы (Сумма / Оплачено / К оплате) + Карточка пациента + История операций

**Команды:** Записать и закрыть | Отметить оплату | Отменить заказ | + Задача по заказу | Печать/Счёт/Чек.

### 4.9 Карточка задачи

**FR-покрытие:** FR-006, FR-009, FR-014.

**Двухколоночная компоновка:**

Левая колонка:
- Шапка с номером, датой, ссылками на пациента и заказ
- Группа «Основные»: заголовок | пациент | ответственный (FR-014: замещающий) | тип задачи | приоритет (radio с цветной полоской) | срок (дата+время+напоминание) | канал | шаблон скрипта | **привязка к заказу** (dropdown)
- Группа «Теги»: мульти-выбор + кастомное добавление
- «Описание / Цель задачи»: textarea
- «Результат выполнения»: чипсы исходов + след. контакт + комментарий

Правая колонка:
- Карточка пациента (компактная)
- Карточка заказа (если привязан) с метриками + список услуг
- История / Аудит

## 6. RBAC — матрица ролей

| Роль | Видит | Не видит | Может |
|---|---|---|---|
| **РОП (Васильев С.И.)** | Все пациенты, все заказы, все задачи, все дашборды, замещения | — | Управлять распределением, эскалировать, деактивировать кураторов |
| **Куратор (Аникина / Романова / Горбунов)** | Только своих пациентов и задачи | Чужих кураторов | Создавать задачи, обновлять статусы, применять стратегию дожима |
| **Помощник куратора (Иванова Н.А.)** | Задачи прикреплённого куратора (без ДМС / Не связываться) | Финансовые суммы, эскалации, чужих кураторов | Закрывать задачи только с комментарием, передавать запрос замены |
| **Администратор (Кузнецова)** | Все настройки, справочники, журналы | Мед. данные пациентов | Управлять ролями, источниками, шаблонами |

При переключении роли в топбаре:
- РОП → автоматически открывается «Дашборд РОП»
- Куратор → «Дашборд куратора»
- Помощник → «РМ помощника куратора»
- Администратор / РОП — все экраны
