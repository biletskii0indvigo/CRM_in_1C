### 1.1 FR-007 · Триггер +3 рабочих дня (GAP P1.3) ✅

**Что:** Кнопка **«Завершить приём (триггер +3)»** в командной панели карточки заказа.

**Поведение:**
- При нажатии система рассчитывает дату +3 рабочих дня (исключая Сб/Вс)
- Создаётся авто-задача куратору с типом «Авто-задача», тегами `+3 дня`, `звонок`, `первичный`, `FR-007`
- Статус заказа переводится в `proposed` (Сделано предложение)
- Появляется зелёный chip «Триггер +3 дня запущен» в карточке заказа
- Защита от повторного запуска: `o._triggered = true`

**Acceptance Criteria (FR-007):** ✓ выполнен — Given первичный приём завершён → When триггер срабатывает → Then у куратора появляется задача с типом «Контакт после первичного приёма» через 3 рабочих дня.

### 1.2 FR-016 · Работа с абонементами (GAP P1.4) ✅

**Что:** Кнопка **«Отметить занятие»** на блоке абонемента в карточке пациента.

**Поведение:**
- Прогресс-бар занятий с цветовым градиентом (зелёный → оранжевый по мере исчерпания)
- Каждое нажатие увеличивает `used` на 1
- На **предпоследнем** занятии (`remaining === 1`):
  - Создаётся авто-задача типа «Авто-задача» с приоритетом High
  - Заголовок: «Продление абонемента: ИФ»
  - Теги: `абонемент`, `продление`, `FR-016`
- На последнем: toast «Абонемент завершён»

**Acceptance Criteria (FR-016):** ✓ выполнен — Given пациент прошёл 11 из 12 занятий → When факт 11-го занятия записан → Then у куратора появляется задача «Продление абонемента: ФИО».

### 1.3 FR-004 · Полные цветные теги (GAP P1.5) ✅

**Что добавлено:**
- Колонка **«Возр.»** в списке пациентов — pill категорий `<30 / 30-50 / 50-70 / 70+` с цветовой кодировкой
- Колонка **«Финанс.»** — pill `ДМС / УМС / Собств.`
- В карточке пациента блок «Атрибуты» дополнен индикаторами возраста и финансирования
- Теги видны без открытия карточки (соответствует требованию FR-004 «куратор / помощник принимает решение по распределению без клика»)

| 50–70 | янтарный |

**Финансирование:**

|---|---|
| ДМС | янтарный |

### 1.4 FR-006 · Обязательный комментарий при закрытии (GAP P1.6) ✅

**Что:** Глобальная модалка `showCloseModal(taskId)` теперь срабатывает при любой попытке закрыть задачу:
- Чекбокс «☐» в списке «Задачи кураторов»
- Кнопка «Выполнить» в карточке задачи
- Любое другое место завершения

**Поведение модалки:**
- Заголовок «Закрытие задачи · комментарий обязателен»
- Info-баннер про FR-006
- Textarea с валидацией (мин. 5 символов)
- Кнопки-чипсы стандартных исходов (быстрая вставка)
- Кнопка «Закрыть с комментарием» disabled пока не введено ≥5 символов
- **Исключение:** задачи с тегом `ортопедия` или типом «Ортопед» — закрываются без комментария (соответствует FR-006 правилу)

**Acceptance Criteria (FR-006):** ✓ выполнен — Given куратор пытается закрыть задачу без комментария → When нажимает «Закрыть» → Then система блокирует и выводит «Введите результат выполнения».

### 1.5 FR-015 · Деактивация куратора с предпросмотром (GAP P1.8) ✅

**Что:** Полноценная модалка `showDeactivationModal(curatorId)` на экране «Замещения».

**Структура модалки:**
- Красный alert-баннер о необратимости
- Правила перераспределения (FR-015):
  - ВИП / Реанимация / Иногородние → **опытному** куратору
  - Первичные / новые → **менее загруженному**
  - Повторные / реактивированные → балансировка
- Полная таблица предпросмотра — все пациенты деактивируемого куратора с указанием нового ответственного и причины
- Сводка по новым ответственным (chip'ы с количеством)
- Кнопка «Деактивировать и распределить» (красная)

**После подтверждения:**
- Все пациенты переназначаются согласно правилам
- Все открытые задачи получают нового куратора
- История кураторов обновляется
- Toast с количеством перераспределённых

**Acceptance Criteria (FR-015):** ✓ выполнен — Given аккаунт куратора деактивирован → When администратор подтверждает → Then открытые заказы равномерно распределяются по правилам.

### 1.6 FR-001 P2.8 · Лимит 2 кураторов на пациента ✅

**Что:** Валидация в карточке пациента при смене куратора (`tryChangeCurator(pid, newCurId)`).

**Логика:**
- В data.js добавлено поле `curatorHistory[]` на каждом пациенте
- При попытке назначить **третьего** куратора — система блокирует и показывает модалку:
  - Красный alert «Превышен лимит»
  - Полная история кураторов пациента (2/2 использовано)
  - Кнопка «Эскалировать РОП» — позволяет принудительное назначение (для исключительных случаев)
- В UI карточки видна история кураторов под флагами с счётчиком «N/2 использовано»

### 1.8 FR-013 · Счётчик нагрузки куратора ✅

**Что:** Добавлен в шапку каждой дорожки куратора на экране «Утреннее планирование».

**Формула:**
- Норма: **8 ч в день × 60 = 480 минут**
- На пациента: **35 минут** (среднее)
- Нагрузка куратора: `N пациентов × 35 мин`

**Визуализация:**
- Текст: «**X ч** / 8 ч норма»
- Прогресс-бар:
  - <80% — зелёный (норма)
  - 80–100% — янтарный (близко к лимиту)
  - 100% — красный + chip «перегрузка»
