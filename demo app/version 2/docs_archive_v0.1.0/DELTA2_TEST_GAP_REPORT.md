| **TC-002** | FR-002 | Тип приёма не обязателен | В `renderOrderCard()` добавить аттрибут `data-required` на select типа приёма и проверку в `orderSave()` |

### 5.1 TC-001 Duplicate detection — детальный flow

```
Текущее поведение прототипа:
  createDeal() → unshift в ST.patients → openDealCard()
  Нет любых проверок.

Целевое поведение:
  createDeal(name, phone, birthDate):
    candidates = ST.patients.filter(p => 
      normalize(p.name) === normalize(name) ||
      (last4(p.phone) === last4(phone) && diffDays(p.birthDate, birthDate) === 0)
    );
    if (candidates.length > 0):
      showDuplicateModal(candidates)  // выбор: открыть существующий / создать второй с обоснованием
    else:
      создать как сейчас
```
