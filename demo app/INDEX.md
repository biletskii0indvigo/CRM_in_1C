## ⛔ Критические замечания

### 1. Это не 1С, это HTML

Прототип сделан как **standalone HTML-приложение** (одна страница + JS), не имеющая никакого отношения к платформе 1С:Предприятие. Не может быть использован как реализация — только как **концептуальная демонстрация для бизнеса**.

### 2. Дублирование документации

`demo app/version 2/docs/` содержит **полную копию** проектной документации из корня `docs/`:

| Документ | Версия в `demo app/version 2/docs/` | Версия в корневой `docs/` |
|----------|--------------------------------------|--------------------------|
| INDEX.md | старая | v0.2.0 |
| TECHNICAL_SPECIFICATION.md | v0.1.0 | v0.2.0 |
| FUNCTIONAL_REQUIREMENTS.md | старая | v0.1.0 |
| Все остальные документы | старые копии | актуальные |

**Плюс уникальные документы только в `demo app/version 2/docs/`:**
- `DELTA1_GAP_CLOSURE.md`, `DELTA2_TEST_GAP_REPORT.md`, `DELTA3_GAP_CLOSURE.md`, `DELTA4_TASK_BP_REVIEW.md` — внутренние отчёты прототипа
- `figma_blueprint_v2.md` — обновлённая версия figma blueprint

### 3. Тяжёлые медиа

`uploads/` содержит **~22 МБ медиафайлов**: 2 версии видео (`0519.mp4`, `0519_h264.mp4`) и 9 PNG-скриншотов. Эти файлы влияют на размер git-репозитория.

## Проблемы

| Проблема | Влияние |
|----------|--------|
| HTML-прототип может быть **ошибочно принят за рабочую систему** | Внешние заинтересованные лица думают «уже работает» |
| Дублирование документации из `docs/` создаёт **два конкурирующих источника истины** | Невозможно понять, какая версия канонична |
| Версии в `demo app/version 2/docs/` уже **разошлись** с корневыми | Чтение устаревших данных |
| Размер папки `uploads/` (~22 МБ) | Раздувает git-репозиторий |
| Нет README, поясняющего «это концепт, не реализация» | Риск неверного восприятия |

## Рекомендации

1. Добавить в корень папки `README.md` с явной пометкой: «**Концептуальный HTML-прототип. НЕ является реализацией. Реализация — расширение .cfe для 1С:Медицина**».
2. **Удалить или заархивировать** `demo app/version 2/docs/` (либо помечать в каждом файле: «архивная копия от 18.05, актуальное — в корневом `docs/`»).
3. Версия 1 — переместить в `demo app/_archive/version_1/`.
4. Видео-файлы (`*.mp4`) — рассмотреть перенос в Git LFS или удаление, если они не нужны постоянно.
5. PNG-скриншоты `f_005…f_155.png` — если это отрисовка концепта, разместить рядом с FR в `docs/`. Если это «кадры из видео» — удалить (видео уже есть).
