## ЭТАП 3: ГЕНЕРАЦИЯ ТЗ
Сформируй документ в следующей структуре:

### 📄 ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Внедрение Яндекс Трекера

**1. Паспорт проекта**
- Цель внедрения
- Границы проекта (что входит / не входит)
- Ключевые стейкхолдеры
- Критерии успеха (измеримые метрики)

**2. Архитектура в Трекере**
```
┌─────────────────────────────────┐
│ ОЧЕРЕДИ (Queue)                 │
├─────────────────────────────────┤
│ • Название | Ключ | Описание    │
│ • Типы задач                   │
│ • Ответственная команда        │
│ • Рабочий процесс (статусы)    │
└─────────────────────────────────┘
[Повторить для каждой очереди]
```

**3. Модель задач**
- Типы задач (Task, Bug, Incident, Feature, Request...)
- Обязательные и опциональные поля для каждого типа
- Шаблоны создания задач (если нужны)
- Правила именования и нумерации

**4. Рабочие процессы (Workflow)**
```
[Диаграмма статусов для каждого типа задач]
Новый → В работе → Ожидает → Решён → Закрыт
         ↓           ↓
      Эскалация   Возврат
```
- Условия перехода между статусами
- Обязательные действия при переходе (комментарий, исполнитель, срок)
- Автоматические триггеры и авто-статусы

**5. Система прав доступа (RBAC)**
| Роль | Видит очереди | Создаёт | Редактирует | Меняет статус | Видит отчёты | Примечание |
|------|--------------|---------|-------------|---------------|--------------|------------|
| [Роль] | [список] | ✓/✗ | ✓/✗ | ✓/✗ | ✓/✗ | [ограничения] |

- Правила видимости конфиденциальных задач
- Ограничения на комментарии/вложения
- Внешние пользователи: уровень доступа

**6. Доски и визуализация**
- Типы досок (Канбан, Спринт, Список, Дашборд)
- Конфигурация колонок для каждой доски
- Фильтры, группировки, сортировки по умолчанию
- Условное форматирование (цветовые индикаторы)

**7. Проектная иерархия**
- Связь: Проект → Эпик → Задача → Подзадача
- Правила наследования полей и сроков
- Кросс-проектная видимость и отчётность

**8. Автоматизация**
```
ПРАВИЛО: [Название]
├─ Триггер: [Событие/Условие]
├─ Действие: [Что выполняется]
├─ Ограничения: [Когда не применять]
└─ Ответственный: [Кто контролирует]
```
[Повторить для каждого правила]

**9. Интеграции**
| Система | Направление | Данные | Метод | Частота |
|---------|-------------|--------|-------|---------|
| [Система] | Входящие/Исходящие | [Что передаётся] | API/Poll/Webhook | [Реально время/по расписанию] |

**10. Отчётность и аналитика**
- Список дашбордов с описанием виджетов
- Периодические отчёты: содержание, получатели, формат
- Ключевые метрики и формулы расчёта

**11. План внедрения**
```
ЭТАП 1 (Недели 1-2): Настройка базовой архитектуры
├─ [ ] Создать очереди и типы задач
├─ [ ] Настроить рабочие процессы
├─ [ ] Протестировать права доступа
└─ [ ] Подготовить инструкции

ЭТАП 2 (Недели 3-4): Пилот с ограниченной группой
...
```

**12. Риски и допущения**
- Технические ограничения Яндекс Трекера
- Организационные риски (сопротивление, нагрузка)
- Компенсационные меры
