# 📋 Статус `/params/` — расширенный бэклог требований

**Создано в рамках приземления документации (`PROBLEM_ANALYSIS.md` §6).**

## Что это означает

Содержимое `/params/` (≈ 7 700 строк) — это **расширенный бэклог требований** (150+ параметров, матрица объектов, черновик метаданных).

⚠️ **Имена объектов 1С из этой папки НЕЛЬЗЯ использовать как канонические.**

## Почему

Есть подтверждённые нестыковки между `/params/` и `/docs/DATA_MODEL.md`:

| Поле / объект | В `/docs/DATA_MODEL.md` | В `/params/` | Канон |
|--------------|------------------------|--------------|-------|
| Справочник пациента | `Справочник.ФизическиеЛица` + БСП | `Справочник.Пациенты` | Будет утверждён после Discovery |
| Префикс объектов расширения | `CRM_X` | смешано, нет правила | `CRM_X` (предложение, утверждается) |
| Кол-во объектов 1С | 17 | 30+ | Будет утверждён после Discovery |
| ABC-сегменты | 5 значений (внутренняя нестыковка) | 5 значений | 3 значения для MVP (§4.2) |

Полный список — `PROBLEM_ANALYSIS.md` §П-3.

## Правила использования

1. ✅ Использовать `PARAMETERS_REGISTER.md` как **источник идей и пожеланий** заказчика.
2. ✅ Использовать матрицу объектов как **карту покрытия** (что упомянуто, что нет).
3. ❌ **Не** использовать как источник имён `Справочник.X` / `Документ.Y`.
4. ❌ **Не** копировать структуру реквизитов в `DATA_MODEL.md` без сверки с типовой конфигурацией.

## Что планируется

После Discovery будет создан документ:

> **`/params/PARAMETER_TO_FR_MAP.md`** — обратная связь «параметр → FR Фазы X», явно показывающая, какие параметры попали в MVP, какие в Фазу 2+, какие в бэклог.
