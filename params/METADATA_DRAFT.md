## Рекомендуемые подсистемы конфигурации

На основании функциональных блоков предлагается следующая группировка:

| Подсистема | Состав |
|------------|--------|
| `Пациенты` | Справочник.Пациенты + сегменты + регистры характеристик |
| `ВоронкаПродаж` | Документ.Сделка, перечисления статусов, правила переходов |
| `КонсультацииИВедение` | Документ.ПервичныйКонтакт, Документ.ПриёмВрача, Документ.ФинальнаяКонсультация, Документ.НазначениеВедения, Документ.ПромежуточнаяКонсультация |
| `Задачи` | Документ.Задача + бизнес-процессы дожима + НСИ-триггеры |
| `Кураторы` | Сотрудники, доступность, нагрузка, передачи пациентов |
| `Коммуникации` | Документ.ВзаимодействиеСПациентом + шаблоны + NPS |
| `НСИ` | Услуги, диагностики, специализации, скидки, причины |
| `Теги` | Справочник.ТегиПациентов + регистры + правила |
| `Возвраты` | Документ.ЗаявлениеНаВозврат + журнал + статусы согласования |

> Рекомендуется максимально использовать **БСП**: «Пользователи», «Задачи и процессы», «Контактная информация», «Файлы», «Дополнительные реквизиты и сведения», «Печать», «Свойства».
