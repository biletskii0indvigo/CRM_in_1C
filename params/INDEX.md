## Точки интеграции с внешними системами

По решению заказчика интеграция не разворачивается в отдельный объект анализа — даётся только перечень точек касания, выявленных из источников:

- **1С:Медицина** — DOC-01 §7.1: 6 направлений обмена (расписание/номенклатура/оплаты/планы/мед.данные/сегменты), протокол REST/COM, авторизация OAuth 2.0/Basic, TLS 1.3.
- **Телефония Комэджик** — DOC-04: подменные номера для определения источника лида.
- **SMS-шлюз / WhatsApp** — DOC-01 §4.2 Приложение Б: шаблоны сообщений сценария дожима.
- **Мессенджер MAX** — DOC-04: единственный мессенджер для входящих обращений.
- **Группа ВК** — DOC-04: канал лидов (поиск + таргет).
- **Сайт компании** — DOC-04: лиды из поисковой выдачи и Яндекс.Директ + форма заказа звонка.
- **Чат-бот** — DOC-02 §2: новый тип касания.
- **Платёжная система / онлайн-касса** — DOC-02 §2: касания «Оплата» и «Возврат оплаты».
- **Сервис отзывов** — DOC-02 §2: касание «Отзыв о работе центра».
- **Лаборатория Гемотест** — DOC-12 Slide 16: внешняя диагностика, отдельный сайт.
- **Геосервисы (Яндекс Карты, 2ГИС)** — DOC-04: источник лидов (только звонки).
- **Медицинские справочники (Продокторов, Сберздоровье)** — DOC-04: источник лидов.

## Ограничения и допущения

- Все 16 источников из `/mnt/user-data/uploads/` прочитаны полностью и являются основой реестра.
- Каждый параметр снабжён точной цитатой из документа-источника в столбце `Раздел/Цитата`.
- Параметры с уровнем уверенности «Средняя» требуют подтверждения заказчика — отмечены поясняющим комментарием.
- Поле `Уточнить` содержит открытые вопросы по каждому параметру для следующей итерации.
- Названия объектов метаданных приведены в стиле платформы 1С (PascalCase, без пробелов).
- В реестре отражены архитектурные конфликты между документами (например, DOC-09 vs DOC-02 по сделкам) — без принятия решения, с явным указанием конфликта в комментарии аналитика.
