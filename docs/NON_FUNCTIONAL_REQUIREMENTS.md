# NON_FUNCTIONAL_REQUIREMENTS — CRM Индвиго

**Версия:** 0.1.0  
**Дата:** 2026-05-18

---

## NFR-001: Производительность

| Требование | Метрика | Метод проверки |
|-----------|---------|----------------|
| Загрузка карточки пациента | < 2 секунды при 1 000+ записях | Нагрузочное тестирование (JMeter / тест в 1С) |
| Открытие рабочего места куратора | < 3 секунды | Ручное тестирование + замеры |
| Формирование дашборда РОП | < 5 секунд | Замер при 10 000+ заказов |
| Создание задачи по триггеру | < 10 секунд от события | Автоматизированный тест |
| Отправка сообщения в мессенджер | < 30 секунд от триггера | Интеграционный тест |
| Одновременных пользователей | ≥ 50 на отделение без деградации | Стресс-тест |

---

## NFR-002: Надёжность и доступность

| Требование | Метрика |
|-----------|---------|
| Доступность в рабочее время (08:00–22:00) | ≥ 99,5% |
| Планируемые технические работы | Ночное окно 22:00–06:00 |
| Потеря данных при сбое | RPO ≤ 1 час |
| Восстановление после сбоя | RTO ≤ 2 часа |
| Транзакционность операций CRM | Все изменения — в транзакции 1С |
| Журнал ошибок | 100% ошибок фиксируются в журнале регистрации 1С |

---

## NFR-003: Безопасность

| Требование | Реализация |
|-----------|-----------|
| Ролевая модель доступа | Роли 1С: Куратор, ПомощникКуратора, РОП, ВрачCRM, АдминистраторCRM, МаркетологCRM |
| Куратор видит только своих пациентов | Ограничение на уровне запросов (ФильтрПоКуратору) |
| Помощник видит только пациентов своих кураторов | Ограничение на уровне рабочего места |
| РОП видит всех пациентов | Без ограничений по курируемым |
| Аудит действий пользователей | Стандартный журнал регистрации 1С + регистр `CRM_АудитCRM` |
| Соответствие 152-ФЗ | ПДн хранятся в 1С:Медицина; согласие на коммуникации — флаг в профиле |
| Маскирование в логах | ФИО, телефоны, адреса не пишутся в открытые логи |
| Шифрование внешних соединений | TLS 1.3 |
| Хранение API-ключей | Константы 1С, доступ только АдминистраторCRM |

---

## NFR-004: Масштабируемость

| Требование | Значение |
|-----------|---------|
| Максимум пациентов в базе | 100 000+ (ограничений нет на уровне архитектуры) |
| Максимум заказов | 500 000+ |
| Максимум задач | 1 000 000+ |
| Максимум пользователей | 200 активных одновременно (на уровне лицензий 1С) |
| Новые центры/филиалы | Добавляются без изменения архитектуры (расширение поддерживает мультиофисность) |

---

## NFR-005: Удобство использования

| Требование | Метрика |
|-----------|---------|
| Обучение куратора работе с блоком «Ведение» | ≤ 30 минут |
| Обучение помощника работе с рабочим местом | ≤ 1 час |
| Количество кликов для создания задачи | ≤ 3 |
| Количество кликов для закрытия задачи с комментарием | ≤ 4 |
| Поддержка русскоязычного интерфейса | 100% |

---

## NFR-006: Совместимость с обновлениями типовой конфигурации

| Требование | Реализация |
|-----------|-----------|
| Расширение не изменяет типовые объекты | Только расширение реквизитов и подписки на события |
| При обновлении типовой конфигурации | Расширение продолжает работу; при конфликте — флаг в расширении |
| Тестирование совместимости | После каждого обновления типовой конфигурации — регрессионное тестирование (4–8 ч) |
| Версия платформы | 1С:Предприятие 8.3.25 и выше |

---

## NFR-007: Журналирование и мониторинг

| Требование | Реализация |
|-----------|-----------|
| Лог интеграций | Регистр `CRM_ЛогИнтеграций`: 90 дней хранения |
| Лог ошибок | Журнал регистрации 1С + регистр ошибок CRM |
| Метрики задач | Регистр `CRM_МетрикиЗадач`: агрегация ежедневно |
| Уведомления об ошибках | Письмо / push администратору системы при критических ошибках |
| Мониторинг регламентных заданий | Встроенный механизм 1С + алерт при пропуске задания |

## NFR-008: Нормативные требования

| Требование | Источник |
|-----------|---------|
| Персональные данные | 152-ФЗ «О персональных данных» |
| Медицинская документация | Стандарты Минздрава РФ, МКБ |
| Врачебная тайна | ФЗ-323 «Об основах охраны здоровья граждан» |
| Автодиктофон | Требует отдельного юридического заключения (OQ-003) |
| Хранение медицинских данных | В типовой 1С:Медицина (CRM не хранит мед. данные напрямую) |
