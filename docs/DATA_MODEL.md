# DATA MODEL — CRM Индвиго

**Версия:** 0.1.0  
**Дата:** 2026-05-18  
**Платформа:** 1С:Предприятие 8.3.25 | Расширение конфигурации (.cfe)  
**Базовая конфигурация:** 1С:Медицина.Поликлиника

---

## Принципы проектирования

1. CRM реализован как расширение (.cfe). Типовые объекты 1С:Медицина не изменяются.
2. Расширение добавляет новые объекты с префиксом `CRM_` или использует подписки на события типовых объектов.
3. Данные пациентов хранятся в типовом справочнике `ФизическиеЛица`. CRM добавляет свои реквизиты расширением справочника.
4. Все объекты в расширении именуются с суффиксом `_CRM` для исключения коллизий.

## 2. ВЗАИМОСВЯЗИ ОБЪЕКТОВ

```
ФизическиеЛица (типовой) ←── CRM_Заказ (1:N)
CRM_Заказ ──────────────────► CRM_Задача (1:N)
CRM_Задача ──────────────────► CRM_Кураторы (N:1 — ответственный)
CRM_Задача ──────────────────► CRM_Кураторы (N:1 — замещающий)
ФизическиеЛица ─────────────► CRM_КуратораПомощника (N:M)
CRM_Заказ ──────────────────► CRM_ИсторияСтатусовЗаказов
ФизическиеЛица ─────────────► CRM_ТегиПациентов
CRM_Задача ──────────────────► CRM_МетрикиЗадач
```

## 4. ОГРАНИЧЕНИЯ И БИЗНЕС-ПРАВИЛА

| Правило | Реализация |
|---------|-----------|
| Максимум 2 куратора на пациента | Валидация при назначении третьего — уведомление |
| Финальная консультация — обязательные поля | Блокирующая валидация в форме документа |
| Задача без комментария не закрывается | Проверка в модуле проведения документа |
| Помощник не видит ДМС-пациентов | Ограничение в запросе рабочего места |
| Помощник не работает с «Не связываться» | Флаг в профиле пациента |
| Абонемент 12 занятий = 2 мес. | Проверка в регламентном задании |
| Абонемент 36 занятий = 6 мес. | Проверка в регламентном задании |
