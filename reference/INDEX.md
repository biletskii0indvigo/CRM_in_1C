## Проблемы

| Проблема | Влияние |
|----------|---------|
| Папка **называется как будто содержит документацию нашей платформы**, но это документация **другой конфигурации 1С** | Архитектурные решения переносятся без верификации |
| Большой объём (33 PDF) — большая часть не нужна для MVP | Команда тратит время на ознакомление с нерелевантным материалом |
| Папка `html presentation/` смешивает разные источники (1С CRM, 1С Медицина, gemini-code) | Непонятно, какие из них актуальны |
| Файлы-дубликаты (`crm_1c_interactive.html` и `(1).html`) | Дублирование |
| Нет указания, **из какой редакции** 1С:CRM эти PDF (`3.0`? `3.1`?) | Разные редакции — разные возможности |

## Рекомендации

1. **Переименовать папку**: `reference/1С CRM/` → `reference/1С CRM ПРОФ (другой продукт) — источник идей/`. Этого достаточно для понимания статуса.
2. Добавить в каждый раздел дисклеймер: «**Это документация 1С:CRM ПРОФ, отдельной конфигурации 1С. Прямой перенос решений в нашу 1С:Медицина-расширение требует анализа.**»
3. Из 33 PDF оставить как обязательные к прочтению только **5–7** ключевых (по концептам, которые мы используем): Канбан, Воронка, Монитор руководителя, Путь клиента, Триггеры. Остальные — backlog.
4. Удалить дубликаты HTML (`crm_1c_interactive (1).html`).
5. Папку `html presentation/` разбить:
   - реальные презентации для заказчика → `presentations/customer/`;
   - концептуальные эксперименты (gemini-code) → `presentations/experiments/`.
