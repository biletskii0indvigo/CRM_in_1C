# Полный анализ бюджета Finance Budget TCO

## 📊 Резюме оценки

На основе предоставленного xlsx-файла `Budget.xlsx` проведён полный анализ стоимости владения (TCO) для системы учёта бюджетных костов на горизонте **5 лет**.

> ⚠️ **Ревизия v1.1:** Проведена переоценка бюджета. Исходная версия содержала нерелевантные параметры медицинской отрасли, задвоенные compliance-статьи и избыточный стек для платформы 1С. Реалистичный TCO снижен с ~16.5 млн до ~5.5–7.0 млн руб.

## 5. Ключевые риски и допущения

### Риски

1. **Инфляционный риск** — операционные затраты растут на 7.5% ежегодно
2. **Рост пользователей** — при расширении базы >20% потребуется пересмотр лицензий 1С
3. **Кадровый риск** — ключевой 1С-разработчик: при уходе стоимость замены ~300,000–400,000 руб.
4. **Scope creep** — расширение модуля (интеграции, BI) добавляет ~350,000–750,000 руб. (см. Roadmap)

### Допущения

- Платформа **1С:Предприятие** уже лицензирована и эксплуатируется в организации
- Серверная инфраструктура существует; дополнительные капзатраты на железо = 0
- Модуль не обрабатывает медицинские персональные данные (ЕГИСЗ не требуется)
- Resource costs не индексируются (пересмотр при продлении контракта с подрядчиком)

## 6. MVP — Модуль учёта бюджетных затрат в 1С

### Концепция MVP

**Цель:** Минимально жизнеспособный продукт на платформе 1С для учёта и контроля бюджетных затрат:
- Быстрый запуск (8–10 недель)
- Нативная интеграция с существующей 1С-инфраструктурой
- Доказательство ценности перед расширением функционала

---

### 6.1 MVP Scope (Must Have)

#### Модуль 1: Базовый учёт затрат
- [ ] Документ «Статья затрат» (дата, категория, сумма, контрагент)
- [ ] Загрузка из Excel/табличного документа 1С
- [ ] Категоризация: Acquisition / Operational / Resource
- [ ] Отчёт «план/факт» с группировкой по периоду и категории

#### Модуль 2: Управление бюджетом
- [ ] Регистр «Годовой бюджет» с лимитами по категориям
- [ ] Контроль лимитов при проведении документов (предупреждение >80%, блокировка >100%)
- [ ] Версионирование бюджетов (черновик / утверждён / архив)

#### Модуль 3: Ролевая модель (штатная 1С)
- [ ] Администратор (полный доступ)
- [ ] Финансовый менеджер (утверждение, отчёты)
- [ ] Руководитель подразделения (ввод, просмотр своих данных)
- [ ] Наблюдатель (только просмотр)

#### Модуль 4: Выгрузки
- [ ] Печатные формы и экспорт в Excel/PDF
- [ ] Email-уведомления при превышении лимитов

## 8. Приложения

### A. Формулы расчёта

**NPV Calculation:**
```
NPV = Σ (Cash Flow_t / (1 + r)^t)
где r = 12% (discount rate), t = year
```

**Inflation Adjustment:**
```
Cost_Year_n = Cost_Year_1 × (1 + inflation_rate)^(n-1)
где inflation_rate = 7.5%
```

**FTE Cost:**
```
Role Cost = FTE_share × FTE_Fully_Loaded_Cost
где FTE_Fully_Loaded_Cost = 1,200,000 руб./год
```
