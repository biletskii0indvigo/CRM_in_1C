### 3.3. Принципы разработки

1. **Расширение (.cfe), без снятия с поддержки типовой конфигурации.** Все объекты — с префиксом `CRM_`.
2. **Тестирование на бэкапе продуктивной БД** — не на продуктиве.
3. **Регрессия после каждого спринта** — типовая 1С:Медицина должна работать без ошибок.
4. **Каждый FR реализуется только после валидации** в `docs/v0.3.0` — no gold-plating.
5. **Pull request review** аналитиком перед каждым слиянием.

## ЭТАП 4: Пилот и приёмка (3–4 недели)

**Период:** август 2026  
**Участники:** 1–2 куратора + PM + QA + IT клиники

### 4.1. Критерии приёмки MVP (Definition of Done)

- [ ] Расширение `.cfe` устанавливается на боевую конфигурацию без ошибок
- [ ] 1 куратор выполнил сценарий «новый пациент → заказ → задача → закрытие с комментарием» (≤ 5 кликов)
- [ ] Ортопедия оформляется как отдельный заказ, не влияет на статус основного
- [ ] Дожим при частичном выкупе: местный — задача +3 дня, иногородний — задача +10 дней
- [ ] Теги отображаются в карточке и фильтруют список
- [ ] Задача прозвона после МРТ без назначения создаётся автоматически
- [ ] Список ботулинотерапии/аджови формируется при пропуске сеанса
- [ ] Базовый отчёт «Заказы в работе» доступен куратору
- [ ] Регрессия типовой 1С: 0 новых ошибок
- [ ] Документация: actualized DATA_MODEL.md, скриншоты управляемых форм, регламент эксплуатации

### 4.2. Пилотный сценарий

1. **Неделя 1:** Установка на тестовую БД; обучение 2 кураторов (2 часа); работа в тестовом режиме.
2. **Неделя 2:** Производственная эксплуатация 2 кураторами; сбор обратной связи ежедневно.
3. **Неделя 3:** Исправление критических замечаний; повторная приёмка.
4. **Неделя 4:** Ретроспектива; решение о расширении на всех пользователей.
